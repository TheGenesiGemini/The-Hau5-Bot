# Guía de Contribución

¡Gracias por tu interés en mejorar este proyecto!

## ¿Cómo contribuir?

1. Haz un fork del repo y crea una rama nueva para tu contribución.
2. Realiza tus cambios, asegurándote de seguir las buenas prácticas de Python.
3. No subas información sensible ni tu archivo `.env`.
4. Haz un Pull Request y describe claramente qué mejoras o cambios hiciste.

## Buenas prácticas

- Usa f-strings para formateo de textos.
- Documenta tus funciones y clases con docstrings.
- Agrega logs en vez de prints cuando sea posible.
- Si encuentras bugs, abre un Issue antes de hacer tu PR.

¡Toda mejora es bienvenida!
